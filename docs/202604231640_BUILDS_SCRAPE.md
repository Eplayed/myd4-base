## 二、抓取方案

- 构筑详情页：`https://www.d2core.com/d4/planner?bd={build_id}`（含完整 equipment 数据）

目标：提取构筑列表结构（排序、筛选参数、每页数量）

从现有 d4_builds_final_v2.json 提取 build_id，批量抓取详情：

- 每个详情页含：完整 equipment 对象、被动技能树、巅峰数据
- **注意：详情页含 JS 渲染，Firecrawl 需要用 `waitFor` 或 `pageOptions.waitFor`**

#### 步骤3：数据合并

## 四、构筑详情→模拟器展示

### 跳转流程
1. 点击构筑卡片 `查看详情` 按钮
2. 跳转到模拟器 tab，加载该构筑数据
3. 模拟器自动填充：
   - 职业选择（class_en）
   - 装备槽（10槽，从 equipment 填充）
   - 技能栏（6格，从 equip_skills 填充）
   - 被动技能（从 passive/数据填充）
   - 巅峰树（从 paragon 数据填充）

### 数据结构（模拟器期望格式）
```javascript
// 构筑数据加载到模拟器
const buildData = {
  classId: 'paladin',           // 职业ID
  equipment: {
    helm: { itemId: 'xxx', ... },
    chest: { itemId: 'xxx', ... },
    // ... 10槽
  },
  skills: [
    { name: 'paladin_falling_star', rank: 1, mods: ['Enhanced...', 'Freefall'] },
    // ... 6格
  ],
  passives: [...],
  paragon: {...}
};
```

### 实现方式
- 模拟器已有 `loadBuild(buildData)` 接口
- 点击构筑卡片时调用 `simLoadBuild(buildData)`
- URL 参数支持：`?bd=1QCR` 直接加载指定构筑
