# 202604241115 - 模拟器页面重构：d2core 风格装备详情 Tooltip

## 目标
将模拟器页面从当前的三栏简单布局重构为 d2core.com 风格的完整构筑查看器：
- 左侧：装备栏（带图标+名称）+ 技能栏（6格）
- 中间：角色展示区 + 装备/符文详情 tooltip（hover 弹出）
- 右侧：符文面板 + 誓约 + 药剂 + 巅峰

## 截图分析

### 截图1 — 头盔 hover 详情
- 大型浮层卡片，显示在装备栏左侧
- 内容：装备名称（金色大字）+ 部位名（灰色小字）+ 物品强度
- 主属性区：护甲值、伤害加成百分比等
- 词缀列表：每条词缀带颜色标记（◆基础 ◆强化 ◆狂暴 ◆威能）
- 威能区域：名称+描述
- 宝石区域：名称+描述
- 底部：掉落 Boss 名单

### 截图2 — 戒指 hover 详情
- 同样的大型浮层卡片格式
- 显示戒指特有属性（主属性、生命上限、冷却缩减、暴击率、伤害加成）
- 威能描述含 [伤害] 形态切换说明
- 宝石信息

### 截图3 — 符文 hover 详情
- 较小的浮层卡片
- 符文名称 + 类型（传奇符文）

## 数据源

| 数据 | 文件 | 字段 | 用途 |
|------|------|------|------|
| 装备列表 | builds_detail_parsed_fixed.json | equipment: {slot: name} | 当前只有名称，需关联 |
| 传奇装备 | uniqueItem_zhCN.json | name, icon, equipType, armor, flavor, dropBoss, affixesDesc | 装备 tooltip 完整数据 |
| 威能 | aspect_zhCN.json | name, icon, affixesDesc, charName, aspectType | 威能 tooltip |

## 实现计划

### Phase 1: 布局重构 (skills-app.js + skills.css)
**目标**: 重构为 d2core 三栏布局

1. **simInjectHTML 重写**
   - 左栏：去掉职业选择卡片（移到顶部或隐藏），装备栏改为垂直列表（每行一个槽位，显示图标+名称）
   - 中间：保留技能列表，但增加装备详情 tooltip 容器
   - 右栏：改为「符文 / 誓约 / 药剂 / 巅峰」四区块

2. **simRenderEquipGrid 重写**
   - 从 2列网格 → 垂直列表（类似截图左侧的装备列表）
   - 每个槽位：装备图标(64x64) + 装备名称 + 部位标签
   - 空槽位显示占位图标 + "空" 标签
   - hover 触发 `showEquipTooltip(slotKey)`

3. **新增 simEquipTooltip 系统**
   - 全局 tooltip DOM 元素（绝对定位，跟随鼠标或固定在装备栏旁）
   - `showEquipTooltip(slotKey)` 函数：
     - 从 SimState.equipment[slotKey] 取装备名
     - 在 uniqueItem_zhCN.json 中查找匹配项（按名称模糊匹配）
     - 渲染 tooltip HTML：名称/部位/物品强度/主属性/词缀/威能/宝石/Boss
   - `hideEquipTooltip()` 函数
   - 同理实现 `showRuneTooltip(runeId)` / `hideRuneTooltip()`

4. **右侧面板重写**
   - 符文网格：2个条件符文 + 2个仪式符文插槽
   - 誓约：1个插槽
   - 药剂：1个插槽
   - 巅峰：5个巅峰节点（修磨/灵力/仲裁宫/使徒/精明）

### Phase 2: 数据关联层
**目标**: 构筑数据 → tooltip 数据的桥梁

5. **创建装备名称索引**
   - 页面加载时解析 uniqueItem_zhCN.json，建立 name→item 映射
   - 支持模糊匹配（因为 builds_detail 中的装备名可能不完整）

6. **tooltip 数据渲染函数**
   - `renderEquipTooltip(itemData)` — 返回 HTML 字符串
   - `renderAspectTooltip(aspectData)` — 返回 HTML 字符串  
   - `renderRuneTooltip(runeData)` — 返回 HTML 字符串

### Phase 3: CSS 样式
**目标**: d2core 暗黑风格 tooltip

7. **新增 tooltip 样式** (skills.css)
   - `.sim-tooltip` — 固定定位浮层，暗色背景，金色边框
   - `.sim-tooltip-header` — 名称（金色/神话红色）+ 部位（灰色）
   - `.sim-tooltip-stats` — 主属性区，菱形标记 + 数值
   - `.sim-tooltip-affixes` — 词缀列表，分类型着色
   - `.sim-tooltip-power` — 威能区块（紫色标题）
   - `.sim-tooltip-gem` — 宝石区块
   - `.sim-tooltip-boss` — 掉落 Boss
   - 动画：fade-in/out 过渡

8. **装备栏样式调整**
   - 垂直列表布局，每个槽位 72px 高
   - 图标 56x56 圆角方形
   - hover 高亮边框
   - 选中/已装备状态特殊边框色

## 关键技术决策

1. **Tooltip 定位方式**: 使用 fixed 定位 + JavaScript 计算位置（相对于触发元素），而非纯 CSS hover（因为内容动态生成）
2. **数据加载策略**: 在 initSimulator 阶段额外加载 uniqueItem/aspect/rune JSON，存入 SimState
3. **名称匹配**: 用 indexOf 模糊匹配，因为 Firecrawl 抓取的装备名可能与标准名有差异
4. **职业选择**: 移到顶部 tab 栏或作为第一行小按钮组，不占独立卡片

### 关键设计细节

1. **装备列表不是纯文字列表** — 每行有图标(56-64px) + 名称 + 部位标签，hover 时整行高亮
2. **角色模型居中** — 装备的威能名称以浮动标签形式显示在角色周围
3. **符文面板紧凑** — 每个符文一行：图标(40x40) + 名称(金色) + 描述(灰色小字)
4. **巅峰节点用圆形图标** — 红色边框 = 已激活，灰色 = 未激活
5. **技能栏固定底部** — 6格等宽，图标 ~48x48
6. **整体配色**：深灰黑背景(#1a1a1a)，金色强调(#d4af37)，暗红/紫色用于传奇/神话
